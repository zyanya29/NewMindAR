Aquí tienes la forma más simple (y segura) de crear un repositorio en GitHub, tanto desde la web como desde tu computadora. Pasos rápidos (desde la web) Entra a GitHub y ve a New repository (botón + arriba a la derecha → New repository). Ponle Nombre, elige Public o Private. (Opcional) Marca Add a README, .gitignore (elige plantilla si aplica) y License. Clic en Create repository. ¡Listo! Si ya tienes un proyecto en tu PC y quieres subirlo Opción típica con HTTPS (no necesitas configurar llaves SSH ahora): # Dentro de la carpeta de tu proyecto git init git add . git commit -m "Initial commit" git branch -M main # Reemplaza USUARIO y REPO por los reales git remote add origin https://github.com/USUARIO/REPO.git git push -u origin main Importante: Si creaste el repo en GitHub con README y tu repo local no lo tiene, primero sincroniza antes de empujar: git pull --rebase origin main git push -u origin main Si prefieres clonar el repositorio vacío y luego trabajar # Reemplaza USUARIO/REPO git clone https://github.com/USUARIO/REPO.git cd REPO # agrega/crea tus archivos git add . git commit -m "Initial commit" git push -u origin main Autenticación recomendada (SSH) — opcional pero cómodo # Crea tu llave (ed25519 recomendado) ssh-keygen -t ed25519 -C "tu_correo@ejemplo.com" # Copia el contenido de ~/.ssh/id_ed25519.pub y agrégalo en GitHub: # Settings → SSH and GPG keys → New SSH key # Prueba la conexión ssh -T git@github.com # Usa la URL SSH como remoto: git remote set-url origin git@github.com:USUARIO/REPO.git Con GitHub CLI (súper rápido) Si tienes gh instalado e inicias sesión con gh auth login: # Crea repo desde tu carpeta actual y súbelo gh repo create REPO --public --source=. --remote=origin --push Archivos útiles al crear el repo README.md: describe el proyecto (qué hace, cómo instalar, usar, etc.). .gitignore: ignora archivos de build, entornos, secretos (elige plantilla por lenguaje). LICENSE: define permisos de uso del código. Errores comunes (y soluciones) non-fast-forward al hacer git push: haz git pull --rebase origin main y luego git push. Usuario o correo no configurados: git config --global user.name "Tu Nombre" git config --global user.email "tu_correo@ejemplo.com" Empujar a rama equivocada: asegúrate de estar en main (git branch -M main). Si quieres, dime tu USUARIO/REPO y te doy los comandos listos para copiar y pegar.
